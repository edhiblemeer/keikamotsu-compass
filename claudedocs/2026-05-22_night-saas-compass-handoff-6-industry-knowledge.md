# 業界 5 業種 × 顧客管理 SaaS 知見スナップショット

night-saas-compass.org の業種別 5 記事 (メンエス / デリヘル / ホストクラブ / キャバクラ / ソープランド) 執筆時に参照する **業界リアル素材集**。

tasteck (旧 no-tel) を 2018 年から 8 年間運用、 6 業種 (メンエス来店 / メンエス出張 / デリヘル / ホスト / キャバ / ソープ / ピンサロ) を顧客として抱える中で見えてきた **業界用語 / 商習慣 / 規制 / 現場ペイン / 顧客分析の鍵 / SaaS 評価軸** を業種ごとに整理。

**前提**:
- エリア軸はゼロ (SaaS は物理位置非依存、 全国どこでも同じ問題)
- 法的な業種区分 (風営法上の分類) ではなく、 **業界実務で使われる呼称** で記述
- tasteck 差別化軸を最後にまとめる (公平比較記事の中で tasteck がどう evaluate されるべきかの示唆として)

## 1. メンエス (メンズエステ、 来店型 + 出張型)

### 業界用語

| 用語 | 意味 | SaaS 側で扱うべきデータ項目 |
|---|---|---|
| **セラピスト** | 施術者 (=キャスト)、 個人事業主契約が主流 | 在籍管理、 シフト、 売上配分、 LTV |
| **指名 (本指名)** | 客が特定セラピストを指名する予約 (=本指名率は店舗の生命線) | 指名種別フラグ、 本指名率 KPI |
| **NG (指名 NG)** | 「この客とこのセラピストは合わない」 という属人情報 (二度と組ませない) | NG 管理 = 来店ブロックレベルの重要データ |
| **メニュー** | 施術コース (60 分 / 90 分 / 120 分) | コース別売上、 滞在時間予測 |
| **オプション** | 延長、 アロマ追加、 マッサージ部位追加 | オプション別単価、 セラピストインセンティブ |
| **完全予約制 vs 即時受付** | 来店型は予約制、 出張型は即時 (デリヘル寄り) | 予約 vs 当日受付の比率 |
| **インターバル** | 客退店後の清掃・準備時間 (=セラピストの稼働率に直結) | スケジュール上のバッファ管理 |
| **新規 / リピート / 久しぶり** | 客の来店回数による分類 | 顧客セグメント、 LTV ステージ |

### 商習慣 (全国共通の業務フロー)

**来店型メンエス** (中・高級店):
- 完全予約制が中心、 電話予約 + Web 予約の併用
- セラピスト数: 1 店舗 20-60 名規模、 入れ替わりが激しい (在籍期間 3-6 ヶ月平均)
- 客単価: 60 分 12,000-25,000 円、 90 分 18,000-35,000 円、 オプション込で 30,000 円超え多い
- 客層: 30-50 代男性会社員 / 経営者、 リピート率 60% 超えで成立する business model
- 受付業務: 来客対応、 メニュー説明、 セラピスト送り出し、 会計、 NG 管理、 売上集計 (1 日 30-50 件捌く)
- スケジュール: セラピストごとに分単位で組む、 インターバル必須

**出張型メンエス** (派遣型):
- 即時受付主体、 電話 + LINE 受注
- ホテル / 自宅派遣、 配車計画必須 (デリヘルに近い運用)
- 客単価: 90 分 18,000-25,000 円、 交通費別
- セラピスト数: 30-100 名規模、 出勤シフト管理が複雑

### 規制

- **風営法 (店舗型性風俗特殊営業)** に該当する場合とそうでない場合あり (施術内容次第、 グレーゾーン業態多い)
- **特商法**: 通信販売的契約に近い、 入会金 / コース料金の明示義務
- **18 歳未満禁止**: 採用 + 客側両方 (年齢確認義務)
- **広告規制**: 性的サービスを暗示する表現 NG (Google 広告 / Meta 広告で BAN 多発)
- **MCA (メンズエステ協会的なもの)**: 業界自主規制、 加盟店は一定の運営ルール遵守

### 現場ペイン (経営者 / 受付の声)

1. **指名 NG が属人化**: 受付の頭の中、 紙ノート、 Excel に分散 → 受付交代時 / 新人 受付教育時に消える → 二度と来てほしくない客を再受け付けする事故が月 2-3 件
2. **セラピスト別の本指名率が出ない**: Excel 集計に 1 人 30 分、 月末まとめで 5h かかる、 結果リアルタイム改善できない
3. **履歴が紙台帳**: 客のアレルギー / 苦手な施術 / 過去トラブル等が紙、 セラピストに事前共有できない
4. **シフト LINE 提出 → Excel 転記**: セラピスト 30 人から「来週シフト」 が LINE で来る、 受付が手動で Excel 集計 = 月 10h
5. **売上計算が複雑**: コース + オプション + セラピスト指名料 + 交通費 + 入会金、 セラピストへの分配 (歩合) も含めると暗算困難
6. **新規流入元分析できない**: ホットペッパー来店 / SNS 来店 / 紹介 来店 の比率が見えない、 広告効果測定できない
7. **採用が常時必要**: 在籍期間短いので毎月採用、 求人媒体課金が経営圧迫 (月 10-30 万)

### 顧客分析の鍵 (経営判断に効く指標)

| 指標 | なぜ重要か |
|---|---|
| **本指名率** | セラピスト個人のスキル / 接客力指標、 60% 超で「指名されるセラピスト」 |
| **LTV (生涯顧客価値)** | リピート顧客 1 人あたり累計売上、 5 万円超えるとロイヤル客 |
| **離脱率** | 来店から次回までの間隔超え (90 日無来店 = 離脱判定) |
| **コース別単価** | 60 分 / 90 分 / 120 分の比率、 高単価コース誘導の指標 |
| **セラピスト別売上 / 稼働率** | 稼働 80% 超え + 本指名 60% 超え = star、 半年後の離職リスクも見える |
| **NG 発生率** | 客側 / セラピスト側両方追跡、 マッチング精度向上の素材 |
| **流入チャネル別 LTV** | ホットペッパー客 vs SNS 客 vs 紹介客 の LTV 差で広告投資判断 |

### tasteck 差別化軸 (中立評価で書く時の参考)

- ✅ **業界 8 年 × 来店型/出張型 両対応** (一般 SaaS は出張型寄りが多い)
- ✅ **指名 NG 自動チェック** (来店受付時に客電話番号入力 → NG セラピスト自動非表示)
- ✅ **セラピスト別本指名率 × LTV × 離脱率 ダッシュボード** (Excel 集計から解放)
- ✅ **スケジュール D&D + インターバル管理** (清掃時間自動挿入)
- ✅ **モバイル 3 系統** (セラピスト用 / 受付用 / 顧客用、 同一プラットフォーム)
- ⚠️ 弱点: ホットペッパー / minimo / OZmall 等の予約サイト直接連携は未対応 (CSV import で代替)、 大手チェーン店は不向き

## 2. デリヘル (デリバリーヘルス、 派遣型)

### 業界用語

| 用語 | 意味 |
|---|---|
| **コール** | 客からの電話受注 (CTI で店舗側に着信) |
| **配車** | 嬢をホテル / 自宅へドライバーが送迎 |
| **着車** | 嬢がホテルに到着 (ドライバーから店舗に報告) |
| **帰還** | 嬢の業務終了 + 店舗 / 待機所への帰投 |
| **写真送り** | 客からの「写真ください」 リクエスト (在籍嬢の写真選択 → 客に送信) |
| **NG (客 NG)** | 「この客には嬢を入れない」 ブラックリスト |
| **折り返し** | 受付電話が混んでて出られない時、 後で店舗から客に電話する運用 |
| **オールタイム** | 24 時間営業 (深夜帯のコール対応必須) |
| **待機所** | 嬢が次のコール待ちで待機する場所 (ホテル / 駐車場 / 店舗内) |
| **本指名 / 写真指名 / フリー** | 客からの嬢選択方法 |

### 商習慣

- 電話受付主体 (LINE 受付も増えてるが、 CTI が中心)
- 配車計画書を毎日作成 (どのドライバーがどの嬢を何時にどこへ運ぶか)
- 客単価: 60 分 16,000-25,000 円、 90 分 22,000-35,000 円、 ホテル代別
- 嬢数: 1 店舗 30-100 名、 出勤シフトはオンラインで 30 分単位
- ドライバー数: 5-15 名、 シフト + 配車スキル管理
- リピート率 90% 超えが成立条件 (新規獲得コストが高い、 リピーター育成が経営の生命線)
- 売掛 (ツケ) 対応の店もあり (常連客向け、 回収管理必須)

### 規制

- **風営法 (無店舗型性風俗特殊営業)** 届出必須
- **特商法**: コース料金 / オプション料金の明示
- **18 歳未満禁止**: 採用 + 客側両方
- **広告規制**: 性的サービス明示 NG (業界専門メディア / 求人サイトに限定)
- **配車禁止区域**: 各都道府県条例で禁止区域あり (学校近接等)
- **本番行為禁止**: 厳密には性風俗特殊営業の範囲を超える行為は違法

### 現場ペイン

1. **電話パンク**: 受付 1 人で同時 3-5 件のコール捌けず、 折り返し率 30%、 機会損失 = 月数十万
2. **配車計画書が紙 / Excel**: 当日朝に手書きで作成、 変更時の連絡漏れで配車事故月数件
3. **嬢別の本指名率 / 売上が見えない**: 月末に手集計、 リアルタイム把握できない = star 嬢の育成失敗
4. **CTI と顧客台帳が分離**: 着信時に客名 / 過去履歴 / NG が即時表示されない = 受付応対品質が個人依存
5. **売掛回収もれ**: 常連の売掛が紙台帳 → 「払ったっけ?」 トラブル + 回収忘れで月数十万損失
6. **写真送り対応 = 全部受付の手作業**: 客から「写真ください」 → 受付が嬢一覧見て選択 → 客に LINE 送信、 1 件 5 分 = 1 日 50 件で 4h
7. **広告効果測定できない**: どの広告 / どのサイト経由で来た客なのかコール時に聞かない (聞きにくい)、 結果リピート広告予算が無駄に膨らむ

### 顧客分析の鍵

| 指標 | なぜ重要か |
|---|---|
| **CTI 連動 LTV** | 着信時に「累計売上 80 万円のロイヤル客」 が出れば応対品質が変わる |
| **嬢別 CPM (Cost Per Mille / customer per minute)** | 嬢の時間効率 + 客単価で算出、 給与査定の根拠 |
| **本指名率** | 写真指名 + フリーから本指名への昇格率、 嬢のリピート獲得力指標 |
| **離脱率 (60 日無来店)** | リピート率 90% モデルのため、 60 日離脱の早期検知が重要 |
| **コース別単価 / オプション率** | 90 分以上 + オプション 3 つ以上の客が high value |
| **売掛残高 / 平均回収日数** | 3 日 / 7 日 / 14 日でエスカレーション、 14 日超は法務介入 |
| **配車エリア別売上** | 客の住所分布、 ドライバー配置最適化の素材 |

### tasteck 差別化軸

- ✅ **業界 8 年 × CTI 連動標準装備** (フルフリー連携実装、 着信時に客カード自動 popup)
- ✅ **配車管理 D&D** (ドライバー × 嬢 × 時間のグリッド、 衝突自動検知)
- ✅ **売掛 3/7/14 日エスカレーション** (回収もれ自動アラート)
- ✅ **嬢別 LTV / CPM 4 象限分析**
- ✅ **モバイル 3 系統** (嬢用 / ドライバー用 / 顧客用)
- ⚠️ 弱点: 写真送り自動化はまだない (今後対応予定)、 CTI ハードウェア (Asterisk 等) は別途必要

## 3. ホストクラブ

### 業界用語

| 用語 | 意味 |
|---|---|
| **ホスト** | 接客従業員 (男性)、 個人売上が全て、 月間ランキング制 |
| **指名** (本指名) | 客が特定ホストを指名 (リピート客は基本指名持ち) |
| **同伴** | ホストが営業前 (18-19 時頃) に客と外食、 出勤前合流で来店 |
| **アフター** | 営業後 (深夜 1-3 時) にホストが客と食事 / カラオケ |
| **シャンパンコール / シャンパンタワー** | 高額シャンパン (10-100 万円) を入れる時の店内パフォーマンス |
| **エース客** | 1 ホストにつき月数百万 / 数千万単位で使う上位客 (1-3 名で売上の 50% 構成) |
| **売掛 (ツケ)** | 客が当日払えない分をホスト個人 (店ではなく) が立て替え、 後日回収する慣行 |
| **回収** | 売掛を客から回収する作業 (ホストの本業の半分は回収) |
| **永久指名 / 飛ばし** | 客が指名ホストを変更する/しない |
| **新規 / 同卓** | 初来店客 vs 既存客同伴 |

### 商習慣

- 営業時間: 20 時 - 翌 5 時 (深夜)
- 客単価: シャンパンなしで 2-5 万 / 卓、 シャンパン含で 10-100 万、 上位客は月数百万
- ホスト数: 1 店舗 20-80 名、 ナンバー (ランキング) 制が事業の根幹
- ホスト評価: 個人月間売上、 本指名率、 同伴本数、 アフター本数
- **売掛が経営最重要課題**: 売上の 30-50% が売掛、 回収まで 30-90 日、 回収率 70-90% (回収不能はホスト個人負担になる店 / 店舗負担の店で分かれる)
- 求人広告投資: 月数百万 (新人ホスト常時採用)
- リピート率: 上位客 80% 超 / 一般客 20-40%
- 客層: 20-40 代女性 (キャバ嬢 / 風俗嬢 / 看護師 / OL / 主婦)

### 規制

- **風営法 (接待飲食等営業 1 号)** 届出必須
- **特商法**: 料金体系の明示義務 (基本料金 / セット料金 / 指名料 / シャンパン)
- **未成年禁止**: 採用 + 客側両方
- **広告規制**: ホスト個人 SNS は本人運用 OK だが、 店舗広告は風営法表示
- **客引き禁止**: 路上での声かけ / キャッチ規制強化中 (歌舞伎町 / 中洲等)
- **悪質ホストクラブ問題**: 売掛強要 / 風俗あっせん等で社会問題化、 監視強化中

### 現場ペイン

1. **売掛が Excel / 紙台帳**: ホスト 30 名 × 売掛 客 300 名 = 9,000 レコードを Excel 管理、 月末集計に経理が 20h かかる
2. **ホスト別 LTV が見えない**: 「このホストの上位 5 客の累計売上」 が即座に出ない、 ホスト育成戦略が感覚運営
3. **指名取引分析できない**: 「ホスト A → ホスト B への飛ばし発生時の売上影響」 を数値化できない
4. **シフト管理 = LINE グループ**: ホスト 30 名のシフトが LINE で毎週来る、 マネージャーが Excel 転記
5. **シャンパンコールの売上記録漏れ**: 営業中の口頭オーダー → 翌朝記入 → 漏れ発生月数件
6. **同伴 / アフター記録が紙**: ホスト個人のメモ、 経営側で客接触履歴が把握不能
7. **本指名率の数値化できない**: 「指名持ち客が来店」 を手動でカウント、 結果ランキング集計に時間
8. **採用効率化できない**: 求人広告 → 面接 → 体験入店 → 入店 のファネルが見えない

### 顧客分析の鍵

| 指標 | なぜ重要か |
|---|---|
| **ホスト別 LTV / 上位客集中度** | ホスト 1 人の売上構成、 「上位 3 客で月売上 70%」 だと依存リスク |
| **売掛残高 / 回収日数 / 回収率** | 経営健全性指標、 回収率 70% 下回ると赤字転落 |
| **本指名率 × 滞在客単価** | ホストの真の価値、 指名持って単価高いのが star |
| **エース客の離脱予測** | 90 日無来店 + シャンパン未注文 で離脱危険、 マネージャー介入のトリガー |
| **同伴 / アフター本数** | 客との接触深度、 ホスト個人 KPI |
| **新規 → 指名昇格率** | 体験入店 客から指名持ちに育つ確率、 マニュアル教育の効果測定 |
| **シャンパン売上構成比** | 高単価商品の依存度、 客層の経済力把握 |

### tasteck 差別化軸

- ✅ **業界 8 年 × ホスト売掛管理特化** (3/7/14/30 日でホスト個人 + マネージャーにエスカレーション)
- ✅ **ホスト別 LTV / 上位客集中度ダッシュボード**
- ✅ **本指名率 × 滞在客単価 4 象限分析** (star / 育成中 / 安定 / 要注意)
- ✅ **シフト LINE グループ → そのまま店内ダッシュボード反映**
- ✅ **モバイル (ホスト個人用) で自分の客 LTV / 売掛 / 離脱予測リアルタイム確認**
- ⚠️ 弱点: 客引き / SNS キャンペーン連携は未対応、 シャンパンコール演出は店内 PA 機材側で完結

## 4. キャバクラ

### 業界用語

| 用語 | 意味 |
|---|---|
| **キャスト** (=キャバ嬢) | 接客従業員 (女性)、 ナンバー (ランキング) 制 |
| **指名** (本指名) | 客が特定キャストを指名 |
| **ヘルプ** | 指名以外のキャストが補助で同卓 |
| **同伴** | 営業前外食からの出勤、 ホストと同構造 |
| **アフター** | 営業後の客とお食事 / カラオケ |
| **シャンパンコール** | ホストと同じ高額シャンパンセレモニー |
| **本指名 / 場内指名** | 来店前指名 vs 来店後指名 (場内ヘルプから昇格) |
| **同卓** | 複数客が同じテーブル (グループ来店) |
| **太客 / 細客** | 客の使う金額による分類 |
| **小計 / セット料金 / 指名料 / VIP** | 料金体系の構成要素 |

### 商習慣

- 営業時間: 19 時 - 翌 1 時 (深夜 1 時規制ある県多い)、 一部 5 時まで
- 客単価: セットなしで 1.5-3 万 / 卓、 シャンパン含で 5-30 万、 太客は月数十万
- キャスト数: 1 店舗 30-100 名、 ナンバー制で月間競争
- キャスト評価: 個人月間売上、 本指名本数、 同伴本数
- 売掛: ホスト比で少なめだが存在 (経営側で慎重管理)
- 求人広告投資: 月数百万 (新人キャスト常時採用、 ナイトワーク求人媒体経由)
- リピート率: 30-50% (太客は 90% 超)
- 客層: 30-60 代男性会社員 / 経営者

### 規制

- **風営法 (接待飲食等営業 1 号)** 届出必須
- **特商法**: 料金体系明示
- **未成年禁止**: 採用 + 客側両方
- **客引き禁止**: ホストと同様
- **22 時以降の女性接客**: 一部地域条例で 18 歳未満は 22 時以降労働禁止 (詳細は労働基準法)

### 現場ペイン

1. **シフト LINE 提出 → Excel 転記**: キャスト 50 名のシフトが LINE で来る、 マネージャーが手集計 (月 10h)
2. **同伴 / アフター記録が紙**: 経営側で客接触履歴把握不能
3. **本指名率の数値化できない**: 「指名持ち客が来店」 を手動カウント
4. **太客の離脱兆候を見逃す**: 30 日無来店で本来アラートすべきが気づかず、 ライバル店に流出月数件
5. **キャスト別の場内指名昇格率不明**: 「場内ヘルプ → 本指名昇格」 の確率がキャストごとに分からない、 育成戦略立たず
6. **シャンパン売上記録漏れ** (ホストと同じ)
7. **採用ファネル管理できない** (ホストと同じ)
8. **客層分析できない**: 「会社員 vs 経営者」 「年齢層別 LTV」 が見えない、 マーケティング根拠なし

### 顧客分析の鍵

| 指標 | なぜ重要か |
|---|---|
| **キャスト別 LTV / 太客集中度** | ホストと同構造 |
| **本指名率 × 場内指名昇格率** | キャストの接客力と育成力指標 |
| **太客離脱予測** (30 日無来店 + シャンパン未注文) | 早期介入で流出防止 |
| **同伴 / アフター頻度** | 客との関係深度、 LTV と相関高い |
| **キャスト別月間売上ランキング** | ナンバー競争の根幹データ |
| **シャンパン売上構成比** | 高単価客の経済力 |
| **客層別 LTV** (年齢 / 職業 / 来店頻度) | 広告ターゲティング根拠 |

### tasteck 差別化軸

- ✅ **業界 8 年 × キャバ嬢 ランキングダッシュボード** (月間 / 週間 / 日次の本指名 / 同伴本数 リアルタイム)
- ✅ **本指名率 × 場内指名昇格率 マトリクス** (育成戦略の数値根拠)
- ✅ **太客離脱予測アラート** (30 日無来店 + 売掛増加でマネージャー通知)
- ✅ **キャストモバイルアプリで自分の指名 / シフト / シャンパン売上確認**
- ⚠️ 弱点: シャンパンコール演出 / 店内音響制御は未対応 (PA 業者側で完結)

## 5. ソープランド

### 業界用語

| 用語 | 意味 |
|---|---|
| **嬢** | 接客女性、 個人事業主契約 |
| **入浴料** | 店舗側に払う基本料金 (法的には「公衆浴場の入浴料」 という建付け) |
| **サービス料 (サ料)** | 嬢個人に渡す追加料金 (法的には嬢個人と客の自由恋愛建付け) |
| **本指名 / 写真指名 / フリー** | 客の嬢選択方法 |
| **NS / NN** | 業界隠語 (Safe vs No Safe 等)、 SaaS 上は表示しない方針推奨 |
| **総額 / コース時間** | 60 分 / 90 分 / 120 分の料金体系 |
| **回転** | 嬢が 1 日に何人客を取るか (1 日 3-6 回が標準) |
| **待機 / 出勤待ち** | 嬢が次の客待ちで店内待機 |
| **総額 vs 入浴料単体** | 客が支払う金額の見せ方、 料金トラブル予防に重要 |

### 商習慣

- 営業時間: 9 時 - 翌 0 時 (24h 営業はほぼなし、 深夜 0 時規制で閉店)
- 客単価: 60 分総額 22,000-45,000 円、 90 分 30,000-60,000 円、 120 分 45,000-80,000 円 (高級店 90 分 8 万超)
- 嬢数: 1 店舗 30-100 名、 出勤シフト 30 分単位
- 受付業務: 来客対応 + 嬢選択 + 入浴料受領 + コース説明 + 嬢への引き渡し
- 受付 = ベテラン中心、 客選別 (危険客排除) の判断力必要
- リピート率: 60-80% (高級店は本指名固定客で安定経営)
- 求人広告投資: 月数百万 (常時採用)
- 客層: 30-60 代男性、 会社員 / 経営者 / 自営

### 規制

- **風営法 (店舗型性風俗特殊営業 1 号)** 届出必須
- **特商法**: 入浴料 / コース料金の明示義務
- **未成年禁止**: 採用 + 客側両方
- **公衆浴場法**: 建付け上「公衆浴場」 営業許可必要
- **NS/NN 表記**: 公式 web / 広告では禁止、 業界専門メディアでも自主規制強化中
- **客引き禁止 / 写真使用同意書**: 嬢の写真 web 掲載は本人書面同意必須

### 現場ペイン

1. **電話パンク**: 受付 1-2 人で 1 日 100 件のコール捌けず、 機会損失
2. **嬢別分析を Excel で頑張ってる**: 月末集計に経理が 10h
3. **回転率 / 稼働率管理が手作業**: 1 日の嬢別稼働 (空き時間 / 客取り回数) が事後集計
4. **NG 客リスト = 紙台帳**: 受付に紙のブラックリスト、 着信時の即時照合できない
5. **CTI と顧客台帳の分離**: 着信時に客名 / 累計売上 / NG 情報出ない = 応対品質が個人依存
6. **売掛回収もれ**: 高級店の常連向け売掛、 14 日超え案件月数件
7. **シフト LINE 提出 → Excel 転記**: 月 10h ロス
8. **写真送り対応** (デリヘル同等): 客から「在籍嬢の写真ください」 → 受付が手動選択 → LINE 送信、 1 日数十件

### 顧客分析の鍵

| 指標 | なぜ重要か |
|---|---|
| **CTI 連動 LTV** | 着信時に「ロイヤル客 / 新規 / 久しぶり」 判別、 応対品質向上 |
| **嬢別本指名率 × 稼働率** | 80%+80% = star、 60%+50% = 育成中 |
| **コース別単価 / オプション率** | 高単価コース誘導の指標 |
| **離脱予測** (60 日無来店) | 高級店ほど早期介入の ROI 高い |
| **新規 → 本指名昇格率** | 嬢個人の接客力指標 |
| **NG 客の自動ブロック率** | 受付応対品質の安定化 |
| **売掛残高 / 回収日数** | 高級店経営健全性 |
| **流入チャネル別 LTV** (業界専門サイト / 紙媒体 / 紹介) | 広告投資判断 |

### tasteck 差別化軸

- ✅ **業界 8 年 × CTI 連動 + 嬢別本指名率分析**
- ✅ **回転率 / 稼働率ダッシュボード** (1 日 / 1 週間 / 1 ヶ月単位)
- ✅ **NG 客自動ブロック** (CTI 連動 で着信時に受付に警告)
- ✅ **売掛 3/7/14 日エスカレーション**
- ✅ **モバイル (嬢用 / 受付用 / 顧客用)** 3 系統
- ⚠️ 弱点: NS/NN 等の業界隠語表示は方針として未対応 (法令遵守優先)、 写真送り自動化は今後対応

## tasteck を公平評価するなら (記事内のランキング表現の参考)

5 業種すべてに対応する SaaS は業界内で稀。 tasteck の **強み** と **弱み** を公平に書く時の素材:

### 強み

1. **業界 8 年運用** (2018-2026) = 業界 SaaS では老舗 ⭐
2. **6 業種対応** (メンエス来店 / 出張 / デリヘル / ホスト / キャバ / ソープ / ピンサロ) = カバレッジ最大級
3. **モバイル 3 系統** (店舗スタッフ用 / キャスト用 / 顧客用) 同一プラットフォーム
4. **個人プレイヤー版** (player.tasteck.tech) があり、 ホスト / キャバ嬢個人の確定申告 CSV / LTV 分析対応
5. **業界特化の現場ペインに細かく対応**: 指名 NG / 本指名率 / 売掛エスカレーション / CTI 連動 / インターバル管理

### 弱み (公平評価では明記すべき)

1. **大手チェーン店向け機能弱い**: 店舗統合管理 / 多店舗予算管理は基本機能のみ
2. **ホットペッパー / minimo / OZmall 等の予約サイト直接 API 連携未対応** (CSV import で代替)
3. **UI デザインは機能性重視** (洗練度では SmartHR 系の最新 SaaS に劣る)
4. **採用 / 求人広告管理は別 SaaS 必要** (求人媒体管理機能あるが、 採用ファネル分析は弱い)
5. **多言語対応なし** (日本国内専用、 訪日外国人接客の店舗には不向き)

### 評価軸での自己採点 (15 項目)

15 項目スコア設計を boost-sys 側 (handoff 2) で確定後に当てはめる想定。 上記強み / 弱みを踏まえると、 業界特化系 SaaS としては上位、 ただし「全方位最強」 ではない (どこかでトレードオフあり)。
